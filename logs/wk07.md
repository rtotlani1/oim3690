### Interaction 1

- **What I asked**: why do we need gitignore?
- **What AI gave me**: 
1. Keep secrets out of the repo
API keys, .env files, passwords — if these get committed, anyone with repo access (or anyone browsing your public GitHub) can see them. Once something's in Git history, it's a pain to fully remove.
2. Skip generated/build files
Stuff like node_modules/, dist/, build/, .next/ — these get regenerated from your source code (package.json, etc.), so committing them is pointless bloat. Anyone can run npm install and get them back.
3. Skip OS/editor junk
Files like .DS_Store (Mac), Thumbs.db (Windows), or .vscode/ settings are specific to your machine, not your project. No one else needs them.
4. Skip logs and caches
Things like *.log, .cache/, __pycache__/ — noise that clutters diffs and has nothing to do with actual project code.
Why it matters practically:

Cleaner repo — git status and diffs only show files that actually matter
Smaller repo size — no bloated node_modules folder in your Git history
Fewer merge conflicts — two people's local build artifacts won't clash
No accidental leaks of credentials
